# 附录

## 附录A 主要模块功能与设计思路

### A.1 用户认证模块

用户认证模块负责平台注册、登录、退出和会话维持等功能。系统通过 Flask Session 保存用户登录状态，并在设备管理、数据监测、命令控制和 AI 报告等接口中进行登录校验。该模块的设计思路是以较低实现复杂度满足当前 Web 平台的访问控制需求。

### A.2 设备管理模块

设备管理模块负责维护设备基础信息、用户归属关系和告警阈值配置。平台通过 `device_uid` 和 `device_secret` 区分不同终端，并通过绑定关系建立用户与设备之间的管理联系。该模块为后续数据查询、命令下发和告警分析提供目标对象。

### A.3 设备网关模块

设备网关模块是 ESP32 与平台后端之间的核心接口模块，主要包括注册、心跳、数据上报、命令拉取和命令回执五个部分。其设计思路是统一采用 HTTP + JSON 的交互方式，在终端侧以较低复杂度实现和平台的稳定通信。

### A.4 数据监测模块

数据监测模块负责设备最新数据展示、历史数据查询和图表可视化展示。该模块的实现重点在于按设备编号和时间范围查询数据库记录，并将结果转换为前端图表所需的数据结构。

### A.5 告警通知模块

告警通知模块根据设备阈值配置判断当前温度是否超限，并在满足条件时通过 SMTP 发送邮件提醒。为避免重复发送，系统加入了冷却时间控制机制。该模块设计的核心思路是用尽量简单可靠的方式实现异常联动功能。

### A.6 AI 分析模块

AI 分析模块根据最近一段时间内的统计数据生成文字分析报告。系统优先调用第三方大模型接口，若接口不可用则回退为本地模板分析。该模块的设计思路是增强平台报告能力，同时保证功能在网络异常条件下仍可使用。

### A.7 BLE 配网模块

BLE 配网模块由浏览器端页面和 ESP32 固件共同组成。浏览器通过 Web Bluetooth 扫描设备并下发 JSON 配置，设备完成参数写入、联网和平台注册，并将状态回传页面。该模块的设计重点在于提升终端首次接入的便利性。

## 附录B 关键符号与变量说明

| 符号或变量 | 含义 |
|------------|------|
| `device_uid` | 设备唯一标识 |
| `device_secret` | 设备身份校验密钥 |
| `server_base` | 设备访问平台所使用的基础地址 |
| `temp_warn_high` | 设备温度告警上限阈值 |
| `AI_API_KEY` | 第三方 AI 接口访问密钥 |
| `AI_MODEL` | 第三方 AI 模型或接入点标识 |
| `recorded_at` | 传感数据采集时间 |
| `command_type` | 平台下发的命令类型 |
| `result_msg` | 命令执行结果说明 |

## 附录C 系统使用说明

### C.1 平台启动

1. 安装 Python 依赖；  
2. 配置 `.env` 文件；  
3. 初始化 MySQL 数据库；  
4. 执行 `python run.py` 启动平台。  

### C.2 用户登录

浏览器访问平台首页后，输入账号和密码登录系统。若已执行初始化脚本，可使用默认管理员账号进行首次登录。

### C.3 设备接入

设备接入可分为两种方式。一种是通过 BLE 配网页完成首次联网；另一种是在固件中直接配置网络参数并调用平台注册接口。接入成功后，设备会在平台侧生成对应记录。

### C.4 监测与分析

登录后可进入监测页面查看实时数据与历史曲线，也可在 AI 报告页面发起场景化分析请求，查看平台生成的分析报告。
