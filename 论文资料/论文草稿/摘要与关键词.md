# 摘要与关键词

## 1. 学校要求

- 中文摘要：300 字以内
- 英文摘要：约 250 个实词
- 中英文关键词均需提供
- 摘要需高度概括研究内容、方法、结果和结论
- 摘要应独立成文，不写大段背景铺垫，不展开论证

## 2. 中文摘要草稿方向

建议控制在 220 到 280 字，结构按四句到六句组织：

1. 研究对象与目标  
2. 系统采用的主要技术  
3. 完成的核心功能  
4. 测试或联调结果  
5. 系统意义与应用价值  

## 3. 中文摘要初稿占位

本文围绕环境监测与设备管理需求，设计并实现了一个基于Web的物联网设备管理平台。系统以ESP32-WROOM-32为终端核心，结合DHT11传感器完成环境温湿度采集，并通过BLE配网与WiFi通信实现设备快速接入。服务器端采用Flask和MySQL实现用户管理、设备管理、数据存储、远程控制和分析报告等功能，前端采用原生Web技术与Chart.js完成监测数据可视化展示。在此基础上，系统进一步实现了温度阈值告警、邮件通知以及第三方大模型辅助分析与本地模板降级机制。测试结果表明，该系统能够完成设备注册、数据上报、图表展示、超温告警、AI报告生成和命令回执等业务闭环，具备较好的完整性与工程应用价值。

## 4. 中文关键词建议

关键词：物联网；Web平台；ESP32；BLE配网；环境监测；Flask

## 5. 英文摘要写作口径

英文摘要不建议逐句硬译，应保持学术表达简洁自然，避免中式直译。后续成稿时应以中文摘要定稿版本为基础重新压缩组织。

## 5.1 English Abstract Draft

This paper designs and implements a Web-based Internet of Things device management platform for environmental monitoring and device management scenarios. The system uses ESP32-WROOM-32 as the terminal controller and employs a DHT11 sensor to collect temperature and humidity data. BLE provisioning and WiFi communication are combined to support convenient device access. On the server side, Flask and MySQL are adopted to implement user management, device management, data storage, remote control and report management. On the client side, native Web technologies and Chart.js are used to present monitoring data visually. In addition, the platform integrates temperature threshold alarms, email notification, third-party large-model-assisted analysis and a local fallback mechanism. Test results show that the system can complete the business loop of device registration, data uploading, chart display, over-temperature alarm, AI report generation and command acknowledgment, which indicates good integrity and practical engineering value.

## 6. 英文关键词建议

Keywords: Internet of Things; Web Platform; ESP32; BLE Provisioning; Environmental Monitoring; Flask

## 7. 使用提醒

- 中文摘要最终提交前建议再人工压到 300 字以内；
- 英文摘要提交前建议再手动统计实词数量，并做一次语言润色；
- 摘要中不宜出现过多“本文首先”“然后”“最后”这类流程化连接词；
- 摘要只写做了什么、用了什么、实现了什么、结果如何，不写大段研究背景。
