# 论文草稿仓

本目录用于持续维护毕设论文 Markdown 草稿，当前主线题目暂定为：

**基于 Web 的物联网设备管理平台设计与实现**

写作口径说明：

- 论文类型按本科工科系统设计类组织。
- 项目主线以真实工程实现为准，不写成纯算法论文。
- AI 分析模块定位为增强功能，不宣称自研核心算法创新。
- 实验数据、截图、部署记录、问题排查、个人分析需后续由作者补充原创内容。
- 参考文献格式暂按占位整理，待学校模板明确后统一调整。

已结合《昆明理工大学论文要求》同步的关键规则：

- 论文题目一般不超过 20 个字；
- 中文摘要控制在 300 字以内；
- 英文摘要控制在约 250 个实词；
- 论文应包含：题目、摘要、目录、绪论、正文、结论、总结与体会、致谢、参考文献、附录；
- 绪论、结论、总结与体会、致谢为独立部分；
- 参考文献不少于 15 篇，且至少包含 1 篇外文文献；
- 需附与课题相关的外文资料原文复印件及不少于 3000 汉字的中文翻译；
- 软件类课题附录中应补充主要模块设计思路、符号变量说明、使用说明、关键程序等内容。

当前文件说明：

- `论文总纲.md`：总题目、章节框架、写作口径、阶段安排
- `摘要与关键词.md`：中文摘要、英文摘要、关键词草稿与字数约束
- `第1章-绪论.md`：研究背景、意义、现状提纲、章节安排
- `第2章-需求与方案.md`：需求分析、可行性分析、方案对比、技术路线
- `第3章-总体设计.md`：系统架构、数据流、数据库与关键流程设计
- `第4章-系统实现.md`：后端、前端、固件、AI、告警模块实现草稿
- `第5章-系统测试.md`：测试环境、功能测试、闭环联调、结果分析
- `结论.md`：研究结果、特色、不足、改进方向
- `总结与体会.md`：个人总结与成长体会
- `致谢.md`：致谢草稿
- `术语表与专业解释.md`：术语统一口径
- `公式与图表清单.md`：建议插图、表格、公式和编号规划
- `参考文献待整理.md`：参考文献占位与整理规范
- `附录规划.md`：软件类附录内容组织方案
- `附录初稿.md`：附录正文初稿
- `外文翻译任务.md`：外文原文与中文翻译任务清单
- `原创材料补充清单.md`：必须由作者本人补充的原创内容清单
- `AIGC控制与二次改写提示.md`：AIGC 比例控制与二次创作提示
