# 结论

本文完成了一个基于 Web 的物联网设备管理平台设计与实现。系统以 ESP32-WROOM-32 和 DHT11 为终端基础，结合 Flask、MySQL 和原生 Web 页面，实现了设备接入、数据上报、监测展示、邮件告警、AI 报告和命令回执等功能。

从最终效果看，这个项目已经基本形成了完整业务闭环。设备可以通过 BLE 完成首次配网，联网后向平台注册并持续上报数据，用户可以在页面查看设备状态和历史曲线，也可以触发告警、生成报告并下发控制命令。相比只完成采集或展示其中一部分的系统，这个平台更强调“从接入到使用”整条链路的完整性。

在实现过程中，项目也暴露出一些真实问题，例如 GPIO12 接线导致运行不稳、`server_base` 配置错误导致设备注册失败、环境变量残留影响 SMTP 配置、AI 接口慢响应影响线上分析等。这些问题虽然增加了开发难度，但也说明系统不是停留在方案层面，而是经过了实际联调和修正。

当然，当前系统仍有不足，比如传感器精度一般、Web Bluetooth 受浏览器环境限制、AI 分析依赖第三方接口、控制实时性仍然一般等。后续如果继续扩展，可以从增加传感器类型、统一 HTTPS 部署、优化弱网补传和升级实时通信方式等方向继续完善。
