# 论文总纲

## 1. 论文基本信息

- 题目：基于 Web 的物联网设备管理平台设计与实现
- 论文类型：本科工科系统设计类毕设论文
- 研究对象：面向环境监测与设备管理场景的 Web 物联网平台
- 项目组成：ESP32 终端、BLE 配网、WiFi 通信、Flask 后端、MySQL 数据库、Web 前端、AI 分析、告警通知

- 中文摘要控制在 300 字以内；
- 英文摘要控制在约 250 个实词；

- 软件类课题附录需补充主要模块功能与设计思路、符号变量说明、使用说明、关键程序等内容。

## 2. 论文核心主线

本文围绕物联网设备在线接入、数据采集、平台展示和远程管理等实际需求，设计并实现了一个基于 Web 的物联网设备管理平台。系统以 ESP32-WROOM-32 作为终端硬件载体，通过 DHT11 传感器采集温湿度数据，并利用 BLE 完成配网、WiFi 完成数据上报和命令交互。服务器端采用 Flask 构建 REST 风格接口，使用 MySQL 对用户、设备、传感数据、命令日志和 AI 分析报告进行持久化管理；前端使用原生 HTML、CSS、JavaScript 与 Chart.js 实现设备管理、监测展示、指令控制和报告查看功能。在此基础上，系统进一步集成阈值告警和第三方大模型辅助分析能力，用于增强平台的环境分析与运维辅助效果。

论文重点不是证明某种复杂算法优越性，而是证明系统从设备接入到平台应用的工程闭环完整可用，即：

`ESP32 采集 -> BLE 配网 -> WiFi 上报 -> Flask 后端处理 -> MySQL 存储 -> Web 可视化 -> 阈值告警 -> AI 报告 -> 远程指令回执`

## 4. 统一写作口径

- 后端技术：Python 3、Flask、PyMySQL
- 前端技术：原生 HTML、CSS、JavaScript、Chart.js
- 数据库：MySQL 8.0
- 终端硬件：ESP32-WROOM-32
- 传感器：DHT11，当前真机接线为 `GPIO27`
- 配网方式：Web Bluetooth 下发 WiFi 参数和 `server_base`
- AI 分析：第三方大模型接口 + 本地模板降级
- 告警方式：温度阈值告警 + 邮件通知 + 冷却时间控制
- 远程控制：平台下发命令，设备轮询接口拉取并执行后回执
