## 一、课题与目标

结合物联网设备管理实际需求，实现 **B/S 架构** 的 Web 管理平台，并融入基础 **智能数据分析**（对接第三方 AI 接口）。对设备进行统一信息建模与管理，支撑运维决策。

**论文撰写注意**：若学院模板中写「Java Web」，需在定稿前与指导教师确认，将「实现技术」统一为 **Flask（Python）** 或与模板一致，避免「论文写 Java、代码用 Python」的答辩风险。

## 二、功能范围（与论文内容对应）

| 模块 | 说明 |
|------|------|
| 用户管理 | 注册、登录、会话校验；未登录禁止访问设备相关接口 |
| 设备信息管理 | 设备注册、列表、绑定关系、基础属性维护 |
| 蓝牙配网 + 绑定 | Web Bluetooth 配网流程可视化（Chrome/Edge 桌面、安全上下文）；**设备侧 `server_base` 须为开发机/服务器在局域网或公网可达的 Base URL**，不得为 `127.0.0.1`/`localhost`（设备上回环不是 PC）；本机用 `http://127.0.0.1` 打开配网页时，平台提供 **`GET /api/server/lan-hint`** 辅助一键填入本机局域网地址（详见 `开发文档.md` §4.5、§7.3）。可选「二维码扫码绑定」与账户关联、身份校验 |
| 实时状态监测 | 设备在线/心跳、最新采集值展示；页面支持自动刷新 |
| 运行数据管理 | 历史数据查询、按时间筛选、入库与补传（设备端离线缓存策略） |
| 远程指令 | 指令下发、执行结果回传、指令日志 |
| AI 分析 | 调用第三方 AI：异常识别、趋势分析；支持机房/家庭/工厂/自定义场景；结果入库，前端图表 + 文字报告展示（支持删除） |
| 数据可视化 | Chart.js 展示趋势与统计 |
| 个人中心 | 用户信息、安全相关基础能力；邮箱/电话维护、温度告警阈值配置 |
| 告警通知 | 温度超阈值时前端弹窗 + 邮件通知，支持最近一次告警状态展示与 SMTP 测试 |

## 三、技术栈（锁定）

- **后端**：Python 3.x，Flask，PyMySQL；MVC：`routes` / `service` / `dao`；RESTful；统一返回 `{"code","msg","data"}`；CORS；参数化 SQL。
- **前端**：原生 HTML5、CSS3、JavaScript；**Chart.js**（指定版本在 `requirements`/说明中写明）；中文界面；统一 `fetch` 封装与错误提示。
- **数据库**：MySQL 8.0，InnoDB，utf8mb4；核心表：`user`、`device`、`device_data`、`command_log`、`operation_log`、`ai_report` 等（含创建/更新时间、合理索引）。
- **硬件**：ESP32-WROOM-32（Arduino）；BLE 广播 → 接收 WiFi 信息 → 断 BLE → 连 WiFi → 注册 → 正常运行；DHT11/DHT22、BH1750 等采集与简单滤波；断线重连与离线缓存（如最近 100 条补发）；指令监听与执行反馈；LED 配网状态提示。

## 四、非功能与安全

- 设备接入须做 **身份校验**，禁止非法设备上报与控指令。
- 接口异常不得崩溃，返回友好中文 `msg`。
- AI 调用独立服务模块：重试、解析、结果持久化。
- **部署**：平台可部署于 **Linux 云主机**（如 Ubuntu + Nginx + Gunicorn），设备经 **公网 HTTP(S)** 访问 `server_base`；与「本机 `127.0.0.1` 开发 + 局域网 IP」并列作为交付形态，细节见 `开发文档.md` **第十章**。

- 已完成 ESP32 + DHT11（GPIO27）真机链路：采集 → WiFi → `/api/device/data` 入库 → Web 监测展示。

- BLE 配网真机化、离线缓存补发、远程指令闭环与回执已完成；**配网页**与文档约定：浏览器安全上下文与设备 **`server_base`** 分离；**`GET /api/server/lan-hint`** 适用于本机回环打开配网页场景；**设备连公网部署平台时**，`server_base` 填 **`http(s)://公网域名或IP`**（与 Nginx 对外端口一致，通常为 80/443）。
- 告警通知：个人中心阈值 → 监测页超温弹窗 → 邮件与状态回显已完成。
- **生产环境**：已在 **Ubuntu 轻量云** 上验证 **Nginx 反代 + Gunicorn + systemd + MySQL** 可对外提供 Web 与设备网关；MySQL `root` 密码与 `.env` 需一致，详见 `开发文档.md` §10。
- **云上稳定性修复**：已完成 `iot-flask.service` 路径修复（203/EXEC），并针对方舟慢响应加入超时与重试配置项（`AI_HTTP_TIMEOUT`、`AI_MAX_RETRIES`）；Gunicorn 运行参数支持长耗时接口。
