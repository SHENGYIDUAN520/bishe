## 三、已完成工作（软件）

### 3.1 架构与规范

- Flask **应用工厂**，**MVC 分层**：`routes` / `service` / `dao`。
- REST 统一返回 `{"code","msg","data"}`；Web 接口 **Session 登录校验**；**CORS** 支持携带 Cookie。
- `开发文档.md`：接口清单、库表约定、蓝图前缀、检查清单；`README.md`：环境与依赖说明。

### 3.2 数据库

- 核心表：`user`、`device`、`device_data`、`command_log`、`operation_log`、`ai_report`。
- 用户表名 `` `user` `` 使用反引号；`password_hash` VARCHAR(512)；默认账号 **admin / 123456**（初始化脚本，答辩后应修改）。
- 演示数据：已绑定设备、待绑定设备、多条 `device_data`、示例指令与 AI 报告（便于无硬件时演示页面）。

### 3.3 后端接口（已实现）

- 认证：注册、登录、退出；用户资料。
- 概览：`GET /api/dashboard/overview`。
- 设备：列表、绑定（手动/蓝牙填充/扫码填充）、改名、删除。
- 数据：最新值、历史分页（支持时间筛选）。
- 指令：下发（类型白名单）、历史列表。
- AI：报告生成（机房/普通家庭/工厂/自定义场景）、报告列表/详情/删除；**火山方舟** OpenAI 兼容调用（`AI_API_KEY` + `AI_MODEL`）；失败或未配置时 **本地统计模板降级**。
- **设备网关** `/api/device/*`：已实现注册/心跳/数据上传/拉指令/回执，可用于真机联调。
- **配网辅助** `GET /api/server/lan-hint`（公开、无需登录）：返回本机建议的 `server_base`（局域网 IP + 当前请求端口），供蓝牙配网页在从 `127.0.0.1`/`localhost` 打开时一键填入设备可访问地址。
- 告警：超温阈值（个人中心可配置）+ 监测页弹窗 + SMTP 邮件通知 + 最近一次邮件状态可视化。

### 3.4 依赖与环境问题处理（日志）

- 增加 **requests**：调用方舟 HTTP 接口。
- 增加 **cryptography**：兼容 MySQL 8 **caching_sha2_password** 认证。
- Windows 虚拟环境需执行 `pip install -r requirements.txt`；**勿将 `.env` 提交仓库**（已配置 `.gitignore`）。
- 新增环境变量加载避坑：`python-dotenv` 默认不会覆盖同名进程环境变量，导致修改 `.env` 后仍读旧值（例如 `SMTP_SENDER` 残留）；已在 `run.py` 使用 `load_dotenv(override=True)`，并要求修改 `.env` 后重启进程。

### 3.5 前端页面（已实现）

- 登录 / 注册；**数据概览**；**设备管理**；**数据监测**（Chart.js）；**远程控制**；**AI 分析报告**；**个人中心**；**蓝牙配网**（Web Bluetooth：扫描/连接/下发 WiFi JSON + 状态 notify；**服务器地址**不得为回环：从 `127.0.0.1` 打开页面时用「自动填入局域网地址」调用 `lan-hint`，非回环页则一键填 `location.origin`；`localStorage` 历史与下发前均拦截回环 URL）。
- 统一 `static/js/api.js`（`credentials: 'include'`）、`nav.js` 导航高亮、`common.css` 样式。

### 3.7 云上生产部署（2026-04-12）

- 已在 **腾讯云轻量云 / Ubuntu** 上完成：**apt** 安装 Python 环境、Nginx、Git、MySQL；项目目录 **`venv` + `pip install -r requirements.txt` + `gunicorn`**；**systemd** 常驻 **`gunicorn -b 127.0.0.1:8000 run:app`**；**Nginx** 监听 80 反代至本机 8000；防火墙放行 **22 / 80 / 443**。
- **MySQL**：`root` 与 **`.env` 中 `MYSQL_PASSWORD`** 需一致；将 `root` 改为 `mysql_native_password` 后 **`sudo mysql` 可能无法无密登录**，可用 **`/etc/mysql/debian.cnf`** 中 **debian-sys-maint** 维护账号或控制台重置密码后对齐应用配置。
- **访问**：公网 **`http://公网IP/`** 可打开平台；**本机浏览器 `127.0.0.1`** 与 **云主机 `127.0.0.1`** 不是同一环回；SSH **本地转发**（`-L`）须在 **个人电脑** 执行且依赖私钥 **`ssh -i *.pem`**，关闭终端即断开转发。
- **BLE 配网**：设备 JSON 中 **`server_base`** 在公网部署场景填 **`43.136.174.98`**（与对外 HTTP 端口一致），勿填 `127.0.0.1`。
- 详细步骤与 Nginx 超时说明已写入 **`开发文档.md` §10**；`README.md` 含蓝牙配网与本机开发说明。

### 3.8 云上联调与稳定性修复（2026-04-13）

- **systemd 服务修复**：`iot-flask.service` 初始指向 `/home/ubuntu/bishe/.venv/bin/gunicorn`，实际部署目录为 `/root/bishe`，导致 `status=203/EXEC`；已改为 `/root/bishe` 并恢复正常运行。
- **AI 分析超时定位**：`/api/ai/analyze` 在云上出现 `CRITICAL WORKER TIMEOUT`。排查确认：方舟接口可达（`curl` 返回 401 说明网络/TLS 正常），小请求可在约 2.6 秒返回 200，但完整分析请求偶发慢响应。
- **超时参数落地**：新增并启用环境变量 `AI_HTTP_TIMEOUT`、`AI_MAX_RETRIES`；当前通过加大 gunicorn `-t` 与 AI 请求超时容忍，确保接口不因 worker 被提前杀死而报「非 JSON」。
- **运行参数现状**：`ExecStart=/root/bishe/.venv/bin/gunicorn -w 2 -t 180 -b 127.0.0.1:8000 run:app`；服务重启与状态检查命令已固化到 `服务器.md`。
- **蓝牙配网下一步结论**：Web Bluetooth 在浏览器安全上下文限制下，本机 `127.0.0.1` 可用，局域网 HTTP 不稳定；已确定下一步方案为 **域名 + HTTPS + Nginx 反代**（纳入计划，暂未实施）。

### 3.6 今日硬件联调进展（2026-04-06）

- 已完成 **ESP32 + DHT11(GPIO27)** 真机烧录与运行，串口可持续读取温湿度。
- 设备可成功调用：`/api/device/register`、`/api/device/data`、`/api/device/heartbeat`，后端返回 `code=0`。
- `device` 与 `device_data` 表已写入真实硬件记录，网页端监测曲线可看到真机数据。
- 修复两项联调问题：
  - **GPIO12 启动干扰**：改为 GPIO27，上传与运行稳定。
  - **注册结果误判**：固件兼容 `"code":0` 与 `"code": 0` 两种响应格式。
- 当前状态：核心链路「传感采集 -> WiFi 上传 -> Flask 入库 -> Web 展示 -> AI 分析」已打通。
