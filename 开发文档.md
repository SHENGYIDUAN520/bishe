> **定位**：本文件为仓库内**唯一权威的工程实施说明**。需求范围以 `开发需求.md` 为准；接口路径、目录结构、命名与数据库字段**以本文件为准**。后续增删接口或表字段时，须**同步修改本文件**，避免前后端与硬件端「各写各的」。

## 一、仓库目录结构（必须遵守）

```
bishe/
├── 开发需求.md              # 业务需求与课题对应（范围级）
├── 开发文档.md              # 本文件：工程规范 + API + 库表 + 目录约定
├── README.md                # 安装、启动、环境变量（操作级）
├── requirements.txt         # Python 依赖及固定版本号
├── .env.example             # 环境变量模板（勿提交真实密钥）
├── run.py                   # 开发环境入口：创建 app 并启动
├── app/
│   ├── __init__.py          # 应用工厂 create_app、注册蓝图、CORS
│   ├── config.py            # 配置类（从环境变量读取）
│   ├── db.py                # PyMySQL 连接获取 / 上下文管理（仅 DAO 层使用）
│   ├── utils/
│   │   ├── response.py      # 统一 JSON 响应封装
│   │   └── auth_helper.py   # 登录会话、设备签名校验等（随开发补充）
│   ├── routes/              # 路由层：参数校验、调用 service、返回 JSON
│   ├── service/             # 业务逻辑层：事务边界、调用多个 DAO
│   └── dao/                 # 数据访问层：仅含 SQL + PyMySQL，禁止写业务判断
├── static/                  # 前端静态资源（CSS/JS/图片）
│   ├── css/
│   ├── js/
│   │   └── api.js           # 统一 fetch 封装（credentials、错误提示）
│   └── pages/               # 各功能页面 HTML（中文界面）
├── sql/
│   └── init.sql             # 建库建表、索引、可选初始数据
└── firmware/                # ESP32 Arduino 工程（后续补充，与硬件接口章节一致）
```

**分层职责**

- **routes**：解析 `request`，必填校验，调用 `service`，**不写 SQL**。
- **service**：组合业务规则、调用 `dao`，**不直接操作 `request` 对象**（可传参）。
- **dao**：`execute` / `fetch` 等，**必须使用参数化占位符**（`%s`），禁止字符串拼接 SQL。

## 四、统一接口规范

### 4.1 HTTP 与路径前缀

- 浏览器调用的业务接口：**`/api`** 前缀（REST 风格）。
- 设备端（ESP32）上报与拉取指令：**`/api/device`** 下独立子路径（见 4.4），与 Web 会话分离，使用**设备身份校验**。

### 4.2 统一响应体

所有 JSON 接口返回：

```json
{
  "code": 0,
  "msg": "操作成功",
  "data": null
}
```

| code | 含义 |
|------|------|
| 0 | 成功 |
| 400 | 参数错误 / 业务校验失败 |
| 401 | 未登录或登录失效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

- `data` 成功时为对象或数组；失败时可为 `null` 或附带简要字段。
- **异常必须捕获**，向客户端返回友好中文 `msg`，**禁止**堆栈信息直接暴露给前端。

### 4.3 用户会话（Web）

- 登录成功后使用 **Flask session**（cookie）维持登录态。
- **设备管理、数据、控制、AI 报告等接口**：除登录/注册及文档明确标注的公开接口外，**必须先校验已登录**，否则 `code=401`。

### 4.4 设备身份校验（硬件）

- 每台设备有唯一 **`device_uid`**（如芯片 MAC 大写无冒号）与 **`device_secret`**（平台生成或注册时返回）。
- 设备调用接口时：请求体或查询参数携带 `device_uid` + `device_secret`（或后续升级为 HMAC，本阶段以明文 secret 传输须在 HTTPS 或内网演示环境说明）。
- 校验失败：`code=403`，`msg` 说明非法设备。

### 4.5 接口清单（随开发实现，路径以本表为准）

**认证与用户**

| 方法 | 路径 | 登录 | 说明 |
|------|------|------|------|
| POST | /api/auth/register | 否 | 用户注册 |
| POST | /api/auth/login | 否 | 登录，写入 session |
| POST | /api/auth/logout | 是 | 退出 |
| GET | /api/user/profile | 是 | 当前用户资料 |

**概览**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/dashboard/overview | 是 | 设备数、今日数据条数、设备简要列表 |

**设备（Web）**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/devices | 当前用户设备列表 |
| POST | /api/devices/demo | 是 | 无硬件时生成一台绑定到当前用户的模拟设备（返回 uid/secret） |
| POST | /api/devices/bind | 扫码/手动绑定（body：device_uid、device_secret 等） |
| PUT | /api/devices/<id> | 修改设备名称等 |
| DELETE | /api/devices/<id> | 解绑/删除（逻辑以 service 为准） |

**数据与监测**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/devices/<id>/latest | 最新一条/心跳状态 |
| GET | /api/devices/<id>/history | 查询参数：start、end、page、page_size |

**指令**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/devices/<id>/commands | 下发指令（类型、参数 JSON） |
| GET | /api/devices/<id>/commands | 指令历史 |

**AI 报告**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/ai/analyze | 触发分析；body 可选 `device_id`；当前实现为**本地统计+文字模板**，可替换为第三方 AI |
| GET | /api/ai/reports | 报告列表 |
| GET | /api/ai/reports/<id> | 报告详情 |

**设备端（硬件，JSON）**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/device/register | 首次注册，分配/校验 secret |

## 五、数据库设计（与 sql/init.sql 一致）

**字符集**：utf8mb4；**引擎**：InnoDB。

| 表名 | 用途 |
|------|------|
| user | 用户 |
| device | 设备主数据与归属用户 |
| device_data | 时序传感数据 |
| command_log | 指令下发与执行结果 |
| operation_log | 用户关键操作审计 |
| ai_report | AI 分析报告 |

**关键字段约定**

- 用户表物理名为 **`user`**，SQL 中一律写作 `` `user` ``（保留字）；`password_hash` 长度 **VARCHAR(512)**，兼容 Werkzeug 多种算法。
- 各表含 `created_at`、`updated_at`（datetime，应用层或 DB 默认值维护）。
- `device.device_uid` 唯一；`device.device_secret` 存哈希或密文（演示阶段可与文档同步说明）。
- `device_data` 对 `(device_id, recorded_at)` 建索引以支持时间范围查询。

详细字段见 `sql/init.sql`；**改表必须同时改 init.sql 与本节文字**。

## 七、硬件与后端契约（摘要）

- 配网流程：BLE 收 SSID/密码 → 断 BLE → WiFi 连接 → 调用 **`POST /api/device/register`** → 正常运行。
- 周期性：**heartbeat**、**data** 上传；轮询或使用长轮询 **`GET /api/device/command`**（后续可演进 WebSocket，毕设阶段以简单方式为主）。
- 离线缓存 100 条补发：设备端固件实现；服务端 **`POST /api/device/data`** 支持批量数组。
