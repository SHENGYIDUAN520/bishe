在你现在这套部署里（systemd + iot-flask.service），更新代码后这样做就行：

重启后端
cd ~/bishe
# 可选：先拉最新代码、装依赖
git pull
source .venv/bin/activate
pip install -r requirements.txt
# 重启服务

Active: active (running) 是否正常
最近日志里有没有报错
实时看日志（最有用）
journalctl -u iot-flask -f
退出实时日志：Ctrl + C

sudo systemctl stop iot-flask      # 停止

sudo systemctl enable iot-flask    # 开机自启

方法二：不想每次登服务器？GitHub 自动部署（WebHook）
适合长期迭代：每次 push 到 GitHub，服务器自动 git pull + 重启服务。

1) 服务器准备自动部署脚本
在服务器项目目录创建脚本 deploy.sh：

#!/bin/bash
set -e

cd /home/ubuntu/bishe
git fetch origin main
git reset --hard origin/main
source .venv/bin/activate
pip install -r requirements.txt
sudo systemctl restart iot-flask

echo "Deploy done at $(date)"

赋权：
chmod +x /home/ubuntu/bishe/deploy.sh

2) 写一个 WebHook 接收服务（推荐单独 systemd）
新建 /home/ubuntu/webhook_server.py：

from flask import Flask, request, abort
import hmac
import hashlib
import subprocess
import os

app = Flask(__name__)
SECRET = os.environ.get("WEBHOOK_SECRET", "")

@app.post("/github-webhook")
def github_webhook():
    signature = request.headers.get("X-Hub-Signature-256", "")
    body = request.get_data()

    if not SECRET:
        abort(500, "WEBHOOK_SECRET not set")

    mac = hmac.new(SECRET.encode(), body, hashlib.sha256)
    expected = "sha256=" + mac.hexdigest()
    if not hmac.compare_digest(expected, signature):
        abort(403, "Invalid signature")

    event = request.headers.get("X-GitHub-Event", "")
    if event != "push":
        return {"msg": "ignored"}, 200

    # 只在 main 分支 push 时部署
    payload = request.get_json(silent=True) or {}
    if payload.get("ref") != "refs/heads/main":
        return {"msg": "not main branch, ignored"}, 200

    subprocess.Popen(["/home/ubuntu/bishe/deploy.sh"])
    return {"msg": "deploy triggered"}, 200

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=9000)

3) 给 WebHook 接收服务配 systemd
创建 /etc/systemd/system/github-webhook.service：

[Unit]
Description=GitHub WebHook Receiver
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu
Environment="WEBHOOK_SECRET=改成你的复杂密钥"
ExecStart=/home/ubuntu/bishe/.venv/bin/python /home/ubuntu/webhook_server.py
Restart=always

[Install]
WantedBy=multi-user.target

启动：
sudo systemctl daemon-reload
sudo systemctl enable github-webhook
sudo systemctl restart github-webhook
sudo systemctl status github-webhook

4) GitHub 仓库配置 WebHook
仓库 -> Settings -> Webhooks -> Add webhook：
- Payload URL: http://你的服务器公网IP:9000/github-webhook
- Content type: application/json
- Secret: 与 WEBHOOK_SECRET 完全一致
- Events: 只勾选 Just the push event

5) 云服务器安全组/防火墙
放行 9000 端口（只需入站 TCP 9000）。
更安全做法：用 Nginx 反代 + HTTPS，并限制来源 IP。

6) 自检命令
查看 WebHook 服务日志：
journalctl -u github-webhook -f

查看后端服务日志：
journalctl -u iot-flask -f
